# EmoFiLM v1 优化基线

已完成数据准备、正式训练、固定生成和本地代理评测的基础 Emo-FiLM 基线领域语言。该上下文只定义后续优化工作使用的当前术语，不继承错误实现或历史诊断分支的语言。

## 基线与资产

**canonical checkout**:
项目唯一受支持的正式工作目录，承载当前活跃主线代码以及本机非 Git 实验资产。其他 checkout、worktree 和历史共享目录都不是后续工作的运行入口。
_Avoid:_ main 工作目录、当前 worktree、共享 checkout

**EmoFiLM v1 优化基线**:
已通过 5-epoch 训练、人工听测、固定 2500 条生成和 `emofilm-eval-v2` 评测的后续优化起点。
_Avoid:_ 临时实验、诊断模型、作者完整复现

**权威基线资产**:
支撑 EmoFiLM v1 继续训练、生成、评测和结果核验的当前数据合同、模型、checkpoint、WAV、manifest、运行身份与指标产物。资产是否纳入 Git 不影响其权威性。
_Avoid:_ 全部未跟踪文件、全部历史实验、可重建缓存

**历史运行身份**:
对一次已经发生的训练、生成或评测所使用输入、代码状态、运行参数、环境和输出关系的事实记录。记录中的历史位置描述当时运行现场，不定义后续运行入口。
_Avoid:_ 当前配置、可执行命令模板、迁移后路径配置

**非权威临时资产**:
不承担当前基线运行或结论证据职责、可安全重建的缓存、符号链接视图和中止运行产物。
_Avoid:_ 正式 full WAV、训练 checkpoint、运行身份

**迁移验收**:
通过资产成员与规模核对、关键加载和正式入口验证，证明 canonical checkout 完整承接权威基线资产并可继续主线实验。
_Avoid:_ 目录复制完成、仅比较总字节数

**遗留共享状态**:
错误 main、历史 debug 或旧实验在共享文件和目录中留下、且未被当前主线白名单选中的内容。它不进入 canonical checkout。
_Avoid:_ 历史证据包、作者原始材料
